# Интерфейс и структура приложения

## Главная таблица референсов

| Референс / Видео | Суб-сценарий | Статус | Креатор | Действия |
|-----------------|--------------|--------|---------|----------|
| Видео 1 | [Выбрать ▼ / Текущий: Сценарий A] | В работе | Алекс | Просмотреть / Редактировать |
| Видео 2 | [Выбрать ▼ / Текущий: Сценарий B] | Готово | Мария | Просмотреть / Редактировать |
| Видео 3 | [Выбрать ▼ / Нет выбранного] | Черновик | Алекс | Просмотреть / Редактировать |

### Логика работы таблицы

1. **Суб-сценарий** — dropdown с 3 вариантами. При выборе открывается форма уточнения параметров (ниша, бренд, тайминг)
2. **Статус** — автоматически обновляется: Черновик → В работе → Готово
3. **Креатор** — назначается при создании сценария или вручную
4. **Действия** — быстрый доступ к просмотру/редактированию сценария

## Экраны и оверлеи

1. **Главная страница** — таблица всех референсов с фильтрами и сортировкой
2. **Авторизация** — вход/регистрация пользователей
3. **Добавление референса** — форма загрузки видео или вставки ссылки
4. **Выбор суб-сценария** — оверлей с 3 вариантами и формой уточнения параметров
5. **Редактирование сценария** — оверлей с текстом сценария и инструментами доработки:
   - Дописать хук
   - Усилить бренд
   - Укоротить/удлинить
   - Упростить/усложнить
6. **Управление креаторами** — список креаторов с поиском и фильтрами
7. **Добавление креатора** — форма создания нового креатора (имя, email, роль)
8. **Настройки таблицы** — фильтры по статусу, креатору, дате; сортировка

## Backend API

1. **Расшифровка видео** — конвертация видео/аудио в текст
2. **Генерация суб-сценариев** — создание 3 вариантов на основе текста референса
3. **Адаптация сценария** — доработка под нишу, бренд, тайминг
4. **Доработка сценария** — применение улучшений (хук, бренд, длина, сложность)
5. **Управление креаторами** — CRUD операции
6. **Управление статусами** — изменение статуса сценария
7. **Назначение креатора** — привязка сценария к пользователю
8. **Экспорт сценария** — выгрузка в различных форматах

## Структура базы данных

- **User** — пользователи системы (имя, email, роль, статус активности)
- **Reference** — референс-видео (ссылка, название, тип: нишевый/общий, дата добавления, расшифровка)
- **SubScenario** — суб-сценарии (текст, связь с референсом, порядковый номер 1-3, выбран ли)
- **Scenario** — финальный сценарий (текст, связь с суб-сценарием, ниша, бренд, тайминг, статус, версия)
- **Assignment** — назначение сценария креатору (связь Scenario ↔ User)
